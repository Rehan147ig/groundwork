Groundwork — System Architecture
Architecture Overview · v1Confidential · June 2026

The authorization layer for enterprise AI.

Groundwork sits at one seam — between the AI and the data — and enforces, at query time, that an agent only ever retrieves what the asking user is allowed to see, and only ever acts where the user is allowed to act. Fail-closed. Fully audited. Below is how the whole system fits together.

02 · Read path

What happens on every query

The agent never talks to the data directly. It asks Groundwork, which verifies, retrieves, checks each chunk against the permission graph, records the decision, and returns only what is authorized.

03 · Two-sided governance

What the agent can see — and what it can do

Retrieval is only half of agent governance. The other half is actions. Groundwork governs both, on the same engine.

Built today

Read side — retrieval

"Can this user SEE this document?"

Every candidate chunk is checked against the permission graph before it reaches the model. Unauthorized, stale, cross-tenant, or region-invalid chunks never enter the prompt.

credit memosKYC packets policy docsboard correspondence audit findings
The second act

Action side — tools

"Can this user, via this agent, DO this?"

RECORDLog every attempted & executed action in the immutable ledger. Do now.
AUTHORIZEPre-action allow/deny via the same engine. v2.
EXECUTENever. The core system executes & does final enforcement.
payment / wire credit-score update loan approval account freeze access grant data export
⚑ credit-score & loan actions = EU AI Act Annex III §5(b) high-risk → mandatory logging + human oversight
Same engine. OpenFGA models user → can_approve → loan exactly as it models user → viewer → document. The hash-chained audit ledger records an action decision exactly as it records a retrieval decision. Actions are an extension, not a rewrite.
04 · The moat

One engine. Connectors per source-type. Packs per vertical.

The enforcement core is identical for every customer and is never forked. Integration is connectors behind one stable interface. Depth compounds into switching cost.
